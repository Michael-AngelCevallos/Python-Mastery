# Engineering Decision #003

## Title

Markdown Over OneNote

---

## Status

Accepted

---

## Context

A permanent documentation system was needed that could live alongside the code.

---

## Decision

All documentation will be written in Markdown files inside the repository instead of OneNote.

---

## Why

Markdown:

- lives with the code
- works with Git
- is portable
- can become a website later

---

## Alternatives Considered

- OneNote
- Google Docs
- Microsoft Word

---

## Consequences

### Positive

✔ Version controlled

✔ Portable

✔ GitHub compatible

✔ Easy website generation

### Negative

• Requires learning Markdown syntax.
