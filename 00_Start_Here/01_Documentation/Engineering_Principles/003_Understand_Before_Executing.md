# Engineering Principle #003

## Principle

Never run a command until I understand what it changes.

---

## Meaning

Every command affects files, folders, or system settings.

Before executing it, I should know what I expect it to do.

---

## Why It Matters

Understanding prevents accidental mistakes and builds confidence.

---

## Example

Before running `git init`, I learned that it creates a hidden `.git` folder and initializes a Git repository.

---

## Key Takeaway

Know what a command does before pressing Enter.
